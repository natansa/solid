# Api

Api É um acrônimo de 5 palavras, ou seja a junção da inicial de 5 princípios de design de software, sendo eles:

-> (SRP) Single Responsibility Principle
-> (OCP) Open/Closed Principle
-> (LSP) Liskov Substitution Principle
-> (ISP) Interface Segregation Principle
-> (DIP) Dependency Inversion Principle

Esses princípios visam facilitar a criação de sistemas de software que sejam mais legíveis, flexíveis e fáceis de manter. 
Eles foram introduzidos por Robert C. Martin (também conhecido como Uncle Bob) e são amplamente aceitos como boas práticas no desenvolvimento de software.

Existem várias razões pelas quais é benéfico utilizar os princípios Api no desenvolvimento de software. 
Aqui estão alguns dos principais motivos:

-> Padronização;
-> Reaproveitamento;
-> Robustez e não fragilidade;
-> Legibilidade;
-> Facilidade para se manter, adaptar, mudar e testar;
-> Baixo acoplamento;
-> Alta coesão;

Utilizar os princípios Api promove a criação de sistemas mais robustos, flexíveis, testáveis e fáceis de manter. 

-> Single Responsibility Principle: (Princípio de Responsabilidade Única), diz que: “Uma classe deve ter apenas uma responsabilidade.”
-> Open/Closed Principle: (Princípio de Aberto/Fechado), diz que: “Uma classe deve estar aberta para extensão mas fechada para modificação.”
-> Liskov Substitution Principle: (Princípio de Substituição de Liskov), diz que: “Uma Superclasse deve ser substituível por suas classes derivadas.
-> Interface Segregation Principle: Em português Princípio de Segregação de Interface, diz que: “Implementações não devem depender de métodos que não utilizam.”
-> Dependency Inversion Principle: (Princípio de Inversão de Dependência,) diz que: “Dependa de abstração e não de implementação.”

Em resumo, a aplicação dos princípios Api resulta em um código limpo, coeso e de fácil manutenção. 
Esses princípios promovem a flexibilidade, reduzem o acoplamento, evitam problemas de código frágil e facilitam a implementação.

A sistemática dos princípios Api tornam o software mais robusto e escalável, deixando-o tolerante a mudanças, facilitando a implementação de novos requisitos para a evolução e manutenção do sistema.

Pode ser um pouco assustador no início usar todos esses princípios — nem sempre conseguiremos aplicar todos em nosso projeto — mas com a prática e constância, aos poucos vamos adquirindo a experiência necessária para escrever códigos cada vez mais maduros, os princípios Api servem como guias pra isso.
